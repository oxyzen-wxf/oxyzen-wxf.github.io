也就是在有异常发生的时候后置通知和异常通知只会有一个进行通知，
假如异常抛出或者未处理一旦发生异常则异常通知会进行通知，而后置通知则失效。

# 🎊@Order 注解

可以使用 @Order 注解指定切面的优先级，值越小优先级越高

@Order 注解，经测试可以使用 @Order 注解指定切面的优先级，值越小优先级越高
假如有两个切面要同时切入同一个方法，可以通过 @Oorder 注解指定优先级，比如同时执行前置通知
那么就先执行前置通知优先级高的也就是值小的哪一个
这里注意一下！！！！
aop 切入我们可以想象相当于一个洋葱，一个平面的洋葱，执行顺序不是优先级低的，在前置通知是晚于优先级高的
前置通知，完事优先级低的后置通知也要晚于优先级高的后置通知，而是早于优先级高的后置通知，这就等同于一个洋葱一个圆里面有好几层，
核心是被切入的方法，最终穿透整个洋葱才算完成织入
