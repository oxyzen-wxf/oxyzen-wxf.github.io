# 模拟 Spring 之 BeanPostProcessor

BeanPostProcessor 也是 Spring 提供给我们的接口，我们可以实现接口来对我们交给 spring 管理的 bean 完成初始化前后的操作。

创建 BeanPostProcessor 接口

接下来我们创建 BeanPostProcessor 的实现类

接下来要做的就是我们可以在 WXFBeanPostProcessor 实现类中的两个方法，对我们的 bean 做初始化前后的操作，两个参数，一个是 bean，一个 beanName 名字，上篇文章我们已经实现了初始化操作，这次就是在初始化前后分别调用我们接口的两个方法，也就是在 createBean 的时候每创建一个 bean 都会调 BeanPostProcessor 的两个方法。

总结：由于只是简单模拟 spring，所以该实现还存在很多不足，就是在创建 BeanPostProcessor 时候我们是自己反射 new 了一个添加进容器，但是这样的话我们在 BeanPostProcessor 实现类中 @autowared 注入其他 bean 的时候则不会被注入，其实应该通过 createBean 创建完成扫描里面有 autowared 注解的类并注入，但是以目前的设计直接 create 创建则会出现在帮 BeanPostProcessor 实现类注入属性的时候，单例池中还没有完成所有 bean 的添加所以还是会注入失败，而要是等所有 bean 都添加完事在来 createBean 创建 BeanPostProcessor 也会有问题，那就是其他 bean 在创建的时候走到 BeanPostProcessor 的两个初始化前后方法时 BeanPostProcessor 还没有被创建，初始化前后方法还是不会走，所以要完成这个功能就需要重新在进行设计一下了。
