# easypoi 导出大量数据

我们平时数据量少的话，直接使用 **ExcelExportUtil.exportExcel ()** 方法就行，但是当我们需要导出的数据在上万条或者几十万条，从查数据库再到导出运行时间就会变得很久，工作中导出算上 sql 时间大约 3w 条数据用了 40 多秒

这个时候我们就可以用 easypoi 的 exportBigExcel 方法了，先来看下官方文档代码

大数据导出是当我们的导出数量在几万，到上百万的数据时，一次从数据库查询这么多数据加载到内存然后写入会对我们的内存和 CPU 都产生压力，这个时候需要我们像分页一样处理导出分段写入 Excel 缓解 Excel 的压力 EasyPoi 提供的是两个方法 强制使用 xssf 版本的 Excel

添加数据和关闭服务，关闭服务不是必须的，可以调也可以不掉 我们只需要 for 循环写入 Excel 就可以了

来看下方法的三个参数：

exportBigExcel (ExportParams entity, Class<?> pojoClass,Collection<?> dataSet) 参数

  • entity: 表格标题属性，设置关于表格信息内容的实体类
  • pojoClass：Excel 对象 Class，我们需要导出数据的实体类
  • dataSet：Excel 对象数据 List，也就是我们需要导出的数据

我们看到我们引的坐标是 springboot 的 starter，所以这里我们在开发中使用的 exportBigExcel 方法的参数可是跟官方文档不同了，这里我们是通过在 easypoi 官网下载实例代码学习 springboot 中的导出大量数据时候的使用方法。

如上代码，参数变成了四个，前面两个参数不变后面的两个参数则与之前不同了

  • entity: 表格标题属性，设置关于表格信息内容的实体类

  • pojoClass：Excel 对象 Class，我们需要导出数据的实体类

  • server：这是一个接口我们可以通过 java8 函数式编程编写，此接口有两个参数

  • queryParams: 查询数据的参数

在导出大量数据的时候 easypoi 会强制导出 xlsx 也就是 07 版本的 excel，07 与 03 版本的区别在于 03 中插入行数是有限制的为 65536 行，256 列，而 07 版本的行数可以大于 100w 行，区别还是很大的，而 easypoi 在导出的时候，一次从数据库查询这么多数据加载到内存然后写入会对我们的内存和 CPU 都产生压力，这个时候需要我们像分页一样处理导出分段写入 Excel 缓解 Excel 的压力。

注意：最好大量数据进行分页处理，每次导出数据最好不要超过 1w 条记录，也就是我们查询数据的数据应该写成分页的方式放到 exportBigExcel 方法中，server 会帮我们循环导入到 excel 效率会更高，这种方式在导出几万条数据的时间也就在几秒时间内.

在使用大量导出方法时候并没有发现导出模板的参数，这个时候就需要我们自定定义导出 excel 的格式了，要不然导出去就是最原始的 excel。这里我们可以通过 ExportParams 对象的 setStyle 方式设置我们自定义的样式。
