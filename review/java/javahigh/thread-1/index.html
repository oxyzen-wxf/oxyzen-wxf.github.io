如果我们还用上面的资源代码逻辑我们有可能会得到如下情况，当然也会出现成功的交替现象，这是由 cpu 的线程调度决定的。

在多线程的世界里我们要了解一件事，那就是要防止虚假唤醒。

先了解下如果我们多个线程操作上面资源类为什么会出现数据没有达到我们想要的结果，其实就是判断出现了问题，所谓的虚假唤醒就是其实并没有到它但是它却醒了做了不该做的事情。= 举个例子 =：当你去做飞机，经过了全面安检，登上飞机，这个时候飞机上的空姐广播说遇见了恐怖分子，将乘客们都劝出了飞机，过会空姐又对大家说，不好意思，刚才收到的通知不属实，还请乘客准备登机。这个时候，乘客应该做的事情是不是应该再过一遍安检才是正确的。这就等同于多线程的虚假唤醒，当四个线程的一个线程拿到资源干完活后通知大家你们可以干活了，然后剩下的三个线程纷纷通过 cpu 调度去争抢资源，谁抢到就是谁的，= 再举个栗子：假如说现在资源在线程一的手中，他干的活是生产者，它干完活为把资源 count 从 0 加到了 1，然后通知其他三个线程，假如说线程二抢到了资源，他也是生产者，但是它是在 if (count != 0) 里面进行 wait，但是一旦他抢到了资源进行唤醒，他就直接干活了，而没有去在判断一下现在是不是应该它干活，就像上面飞机的例子，它应该再过一遍安检，才不会做一些对不起其他人的事情，所以解决的办法就是应该把它拉回来你在判断一次是不是应该你干才对。

所以在多线程资源类中的判断我们都应该使用 while 来进行判断！！！

这样就又达到了我们想要的效果了！
